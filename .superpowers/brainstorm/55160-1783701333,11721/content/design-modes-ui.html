Design §4 — Modes, setup, HUD & win flow

MatchConfig is the single source of truth for a race session.

MatchConfig (Autoload)

enum Mode { HYBRID, RACE, LAST_STANDING }
mode: Mode = HYBRID
lap_count: int = 5          # ignored when LAST_STANDING
ai_count: int = 4           # fixed in v1; easy to expose later
track_scene_path: String    # default Figure8; multi-map later

Setup.tscn (main menu / pre-match)

Setup
PULSE DEATHRACE
MODE: [ Hybrid | Race | Last Standing ]
LAPS: ◀ 5 ▶ (hidden/disabled if Last Standing)
START
  • START writes MatchConfig → get_tree().change_scene_to_file("Main.tscn")
  • Lap options: e.g. 3 / 5 / 7 (or spinbox 1–99); default 5

HUD (during race)

  • Match timer (elapsed)
  • Player lap current / total (hidden in Last Standing)
  • Player HP (numeric and/or bar) — also bar above car
  • Cars remaining (alive count)
  • Mode label (optional, small)

Win / lose rules

ModePlayer winsPlayer loses
Last StandingOnly living car left is playerPlayer dies
RacePlayer finishes required laps firstPlayer dies or an AI finishes first
HybridRace finish first or last standingPlayer dies or AI finishes race first

On end: pause physics / freeze cars → show EndScreen (“You Win!” / “Game Over”) → buttons: Rematch (reload Main) · Setup (back to menu).

Main.gd responsibilities

  • Instance track, spawn 1 player + 4 AI at markers
  • Connect died / lap signals → evaluate win conditions
  • Drive HUD; own or reparent Camera2D on player
