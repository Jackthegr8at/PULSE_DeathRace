Design §3 — Figure-8 track, laps & AI pathing

Built for chaos now; more maps later = drop-in track scenes.

Figure8.tscn (top-down schematic)

Track scene contents

  • Walls: StaticBody2D + CollisionPolygon2D (outer bounds + infield islands). Visuals: Polygon2D or ColorRect placeholders
  • RacePath: Path2D / Curve2D through both loops (one continuous race line through the X)
  • SpawnPoints: Marker2D nodes (5) near start — Main places player + 4 AI
  • Checkpoints: Area2Ds along the path (ordered). Prevents reverse-lap cheese
  • StartFinish: Area2D; lap++ only if all checkpoints cleared this loop
  • Track.gd API: get_path(), get_spawn_transforms(), checkpoint count — maps stay swappable

Laps (when mode uses them)

  • Each car tracks: current checkpoint index, laps completed
  • Race win: first car to reach MatchConfig.lap_count after valid S/F crossing
  • Last Standing mode: lap UI hidden / counters inactive

AI pathing (basic but playable)

  • Progress along Path2D with look-ahead offset (export: path_look_ahead)
  • Steer toward target point; throttle high on straights, ease near sharp curve samples
  • Combat: pick nearest living car in forward cone + range → try_fire on cooldown
  • No advanced avoidance in v1 (bumping OK — chaos road)
  • Optional light rubber-band later — not v1
