Design §1 — Project structure

Approach 2: shared car, track scenes, match config. Confirm this layout before we go deeper.

PULSE_DeathRace/
├── project.godot
├── scenes/
│   ├── Setup.tscn          # mode + laps → START
│   ├── Main.tscn           # race root (Node2D)
│   ├── ui/
│   │   ├── HUD.tscn        # timer, laps, HP, alive count
│   │   └── EndScreen.tscn  # You Win / Game Over
│   ├── cars/
│   │   ├── Car.tscn        # shared CharacterBody2D
│   │   └── Missile.tscn    # Area2D projectile
│   └── tracks/
│       └── Figure8.tscn      # walls + Path2D + spawn points
├── scripts/
│   ├── autoload/
│   │   └── MatchConfig.gd  # mode, lap_count, apply from Setup
│   ├── cars/
│   │   ├── Car.gd          # physics, health, shoot, explode
│   │   ├── PlayerCar.gd    # input → car API
│   │   ├── AICar.gd        # path follow + shoot
│   │   └── Missile.gd
│   ├── tracks/
│   │   └── Track.gd        # spawn points, path, lap checkpoints
│   ├── ui/
│   │   ├── Setup.gd
│   │   ├── HUD.gd
│   │   └── EndScreen.gd
│   └── Main.gd             # spawn cars, win checks, camera
└── docs/                   # design specs (later)

Why this shape

  • New map = new scene under tracks/
  • Modes/laps live in MatchConfig, not hardcode
  • Car logic once; player/AI only control intent
  • Clear extraction points for option-3 components later

Phase 1 limits

  • One track (Figure-8) only
  • Placeholder shapes, no art pipeline
  • No blockchain / XPR
